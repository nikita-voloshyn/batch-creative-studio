# GeminiProvider

## Purpose
`ImageProvider` adapter for Gemini 2.5 Flash Image ("Nano Banana") over the `@google/genai` SDK. Generates one image-conditioned post per item and maps SDK/API errors to neutral `ProviderError` facts.

## Public Interface
- `createGeminiProvider(): ImageProvider` — build the adapter (`id: "gemini"`, `supportsImageReference: true`). Throws `ProviderError("auth", ...)` if `GEMINI_API_KEY` is absent (the registry guards against this).

## Inputs and Outputs
- `generate(input, signal)`:
  1. Fetches the product image + already-normalized references as inline base64 parts via `fetchImageAsInlineData` (references arrive as `data:` URLs, so no extra network).
  2. Calls `ai.models.generateContent` with `contents` = `{ text: prompt }` + one `inlineData` part per image, and `config` = `responseModalities: ["IMAGE"]`, `imageConfig.aspectRatio`, `seed`, `abortSignal`.
  3. Extracts generated bytes from the first candidate's `inlineData.data` (base64).
- Returns `GenerateResult` with `imageBytes`, `usedImageReference = referenceImageUrls.length > 0`, `contentType` = the model-reported MIME, `meta { latencyMs, model }`.
- Error mapping: prompt `blockReason` or content-policy `finishReason` (SAFETY, PROHIBITED_CONTENT, IMAGE_SAFETY, RECITATION, BLOCKLIST, SPII) → `content_policy`; 200-with-no-image → `server`; abort/timeout → `timeout`; HTTP status refined via `refineKind`; network/unknown → `server`.

## Dependencies
- `@google/genai` — `GoogleGenAI`, `ApiError`, content/part types.
- `lib/providers/config.ts` — `geminiApiKey`, `geminiModel`.
- `lib/providers/errors.ts` — `ProviderError`, `kindFromHttpStatus`, `ProviderErrorKind`.
- `lib/providers/reference-normalize.ts` — `fetchImageAsInlineData`.
- `lib/providers/types.ts` — `GenerateInput`, `GenerateResult`, `ImageProvider`.

## Key Decisions
- `refineKind` heuristics: a 429 naming "daily"/"per-day" → `quota_exhausted` (fatal) vs plain rate_limit (retryable); a 400 naming safety/policy → `content_policy`; "unavailable"/"overloaded" → `unavailable`.
- A 200 response with no image bytes is treated as a transient `server` anomaly so the engine retries then fails over, rather than failing the item.
- `retryAfterMsOf` parses a server-suggested delay out of the SDK message text (e.g. `retryDelay: 12s`).
- Keeps private copies of `isAbortError`/`errorMessage`/status extraction because Gemini errors flow through the SDK's `ApiError`, not raw HTTP.

## Known Limitations
- Quota-vs-rate-limit classification leans on message wording (Gemini surfaces both as 429 + RESOURCE_EXHAUSTED).
- Aspect ratio is applied natively via `imageConfig` and also instructed in the prompt text; no post-process enforcement here.

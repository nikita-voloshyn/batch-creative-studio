# Decisions

Accepted decisions that constrain future work. One dated bullet per decision. Append-only.

- 2026-06-26 — **No authentication.** The app is single-user; login / multi-user isolation is deferred to full-product scope. (README §9 / TZ §15)
- 2026-06-26 — **Provider failover chain:** Gemini 2.5 Flash Image ("Nano Banana", primary, native image-reference conditioning) → Cloudflare Workers AI (secondary, free via API) → Replicate (optional tertiary, config-flagged, off by default). `fal.ai` excluded — its free credits do not work via API; OpenAI `gpt-image-1` excluded — no working free tier.
- 2026-06-26 — **Per-image caption hints** live as `Job.params.perImageHints?: Record<string, string>` keyed by `productImageUrl`, resolved at the orchestrator/composition-root when building each Item's `GenerateInput`. There is **no** `Item.captionHint` field.
- 2026-06-26 — **Concurrency knob** is the env var `POOL_SIZE` (default 4–6).
- 2026-06-26 — **Result Blob key** is per-item, attempt-independent: `results/{jobId}/{itemId}.{ext}` ({ext} from content-type), **last-writer-wins** — an item never yields more than one distinct result object and there are no orphan result blobs. The provider-call de-dup idempotency key `hash(itemId + attemptNumber)` is a separate concern and is retained.
- 2026-06-26 — **MVP runtime:** in-memory job/item/attempt state + in-flight orchestration inside the streaming Route Handler (Vercel Fluid Compute). The Terminal invariant (every item/job reaches a terminal state) holds **only within one function lifetime**. A durable queue + Postgres/Neon + KV/Upstash persistence are deferred to full-product scope.
- 2026-06-26 — **Attempt cap is per provider** (default 3 = 1 initial + 2 retries, `attemptNumber ∈ {0,1,2}`); reaching the cap triggers failover. Per-attempt timeout `ATTEMPT_TIMEOUT_MS` (default 60000); a timeout is a retryable error.
- 2026-06-26 — **Toolchain:** pnpm · Vitest · Biome · Next.js App Router on Vercel.
- 2026-06-28 — **Reference style transfer via image stitching (HF Kontext).** ⚠️ SUPERSEDED by the vision-extraction decision below (2026-06-28). FLUX.1-Kontext (via the HF image-to-image API) is single-image, so the reference image's *mood* can't be passed as a second input directly. First solution: the HF adapter stitched the product + reference image(s) side-by-side (jimp) into one input image. This proved UNRELIABLE in live testing — Kontext intermittently ignored the reference (output ≈ input), echoed the reference's objects into the result, or returned a collage / second object in the corner. Replaced by the text-extraction path.
- 2026-06-28 — **Reference MOOD transfer via vision-to-text + product-only Kontext** (live-validated; replaces stitching). Because Kontext is single-image, the reference's mood is extracted ONCE per job by a HuggingFace chat-completion **VLM** (`lib/providers/style-extract.ts`, default `google/gemma-3-27b-it` with Qwen2.5-VL / Llama-3.2-Vision fallbacks; same `HF_TOKEN`, a cheap text call) into a short mood description (lighting / color grade / setting / atmosphere, explicitly NOT naming objects). The orchestrator threads it through `buildPrompt({ referenceStyleText })`, and the HF adapter now sends **only the product** image and re-lights it to that mood by prompt. Result (live-verified: white lamp → reference's teal-and-orange cinematic mood): product preserved exactly, reference mood applied consistently across the batch, and ZERO reference-leak (no second object, since nothing but the product is in frame). HF `supportsImageReference=false` now (the image call conditions on the product only); `GenerateResult.usedImageReference` = "the job had a reference whose mood shaped this output". `jimp` removed (it was only the stitcher). Cloudflare (FLUX.2 multipart) still image-conditions on the reference as a fallback and also receives the text cues. Note: Cloudflare's free 10k-neuron/day cap is easily exhausted, so it is a weak fallback — HF is the reliable path. Why not Gemini "Nano Banana" (native multi-image): free image-gen limit is 0 (needs billing, user declined). A paid IP-Adapter/Redux model would give pixel-exact reference matching as a drop-in upgrade.
- 2026-06-28 — **Product-preserving free path = HuggingFace FLUX.1-Kontext-dev (img2img)** (live-test driven). Pollinations free `gptimage` turned out to be text-to-image (ignores the input image → product lost; same per-batch seed → identical outputs). HuggingFace Inference Providers serves **FLUX.1-Kontext-dev** — a TRUE image-edit model that PRESERVES the input and restyles per the prompt (live-verified: an exact scene was kept, only the art style changed) — via the `fal-ai` provider, covered by HF's small free monthly Inference-Providers credit. Provider `lib/providers/huggingface.ts` (uses `@huggingface/inference` SDK), gated on `HF_TOKEN` (fine-grained token w/ "Inference Providers" permission). Kontext is **single-image**: input = the product (preserved); the reference's style is carried by the prompt/brief (`usedImageReference=false`). Free chain: `PROVIDER_CHAIN=huggingface,pollinations,cloudflare`; `POOL_SIZE=1` (serial — Kontext ~10s + cold-start). When the free HF credit is exhausted (402) → failover to pollinations/cloudflare. The architecture swaps providers via one env var, so a paid img2img key later is a drop-in upgrade.
- 2026-06-28 — **Free provider path = Pollinations `gptimage`** (superseded by HF Kontext above; kept as a fallback). Gemini 2.5 Flash Image has NO free tier (`limit: 0`, needs billing); Cloudflare FLUX.2 klein echoes the reference (poor multi-image composition). Added a **Pollinations provider** (`lib/providers/pollinations.ts`, `image.pollinations.ai`, `Authorization: Bearer POLLINATIONS_TOKEN`, free registration at enter.pollinations.ai) using model **`gptimage`** — the image-input model that works with the free token (verified: 200 + uses the input image). `nanobanana`/`kontext`/`seedream` are premium-gated on enter.pollinations.ai (500 even with the free token) — left configurable via `POLLINATIONS_MODEL` but not default. Free failover chain: `PROVIDER_CHAIN=pollinations,cloudflare` (Gemini dropped from the default chain — free=0 just wastes a 27s retry before failover).
- 2026-06-28 — **Blob store must be PUBLIC** (not Private). Images need public URLs (tiles render them, providers fetch product/reference by URL, export downloads). A Private store rejects `access:"public"` writes. Also: Vercel Blob now defaults to OIDC — for local dev use a static `vercel_blob_rw_` token from the store's `.env.local` snippet.
- 2026-06-28 — **Upload route uses NO `onUploadCompleted` callback** (live-test fix). `app/api/uploads/route.ts`'s `handleUpload()` deliberately omits `onUploadCompleted`: registering it forces `@vercel/blob` to embed a callback URL in the minted client token, which it cannot determine on localhost ("no callbackUrl could be determined") — breaking dev uploads. The MVP needs no callback (in-memory state; `upload()` returns the durable `.url` directly). Do not re-add it without setting `VERCEL_BLOB_CALLBACK_URL` / a `callbackUrl`.
- 2026-06-26 — **Secrets / env management.** Secrets live in `.env.local` (gitignored) for local dev and as Vercel project env vars in production (synced via `vercel env pull .env.local`). `.env.example` is the tracked template documenting every variable. MVP-required: `GEMINI_API_KEY`, `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`. `BLOB_READ_WRITE_TOKEN` is provisioned when the Vercel Blob store is connected (T3/T13). `REPLICATE_API_TOKEN`, `DATABASE_URL`, `KV_URL` are full-product/deferred. Orchestration tunables (`POOL_SIZE`, `ATTEMPT_CAP`, `ATTEMPT_TIMEOUT_MS`, `PROVIDER_CHAIN`, `MAX_ITEMS`, `GEMINI_RPM`, `GEMINI_DAILY_QUOTA`) are env-driven and change without redeploy.
